# 守护长者 - 寻找失踪老人系统 部署说明

## 项目结构
```
ShiZongLaoRen/
├── database/          # 数据库SQL脚本
├── backend/           # SpringBoot 后端（Java 22）
└── miniprogram/       # 微信小程序前端
```

---

## 一、数据库初始化

```sql
mysql -u root -p < database/init.sql
```

---

## 二、后端部署

### 1. 修改配置文件
编辑 `backend/src/main/resources/application.yml`，替换以下配置：

| 配置项 | 说明 |
|--------|------|
| `DB_USERNAME` / `DB_PASSWORD` | MySQL用户名密码 |
| `WECHAT_APP_ID` / `WECHAT_APP_SECRET` | 微信小程序AppID和Secret |
| `ALIYUN_ACCESS_KEY_ID` / `ALIYUN_ACCESS_KEY_SECRET` | 阿里云AccessKey |
| `ALIYUN_OSS_BUCKET` | 阿里云OSS存储桶名称 |
| `ALIYUN_FACE_DB` | 阿里云人脸库名称 |

### 2. 阿里云准备工作

**OSS（对象存储）：**
1. 登录阿里云控制台 → OSS → 创建Bucket
2. 设置读写权限为"公共读"（照片需要公开访问用于人脸识别）
3. 记录Bucket名称和Endpoint

**人脸识别（FaceBody）：**
1. 开通阿里云人脸识别服务
2. 进入控制台 → 人脸识别 → 人脸库管理 → 创建人脸库
3. 记录人脸库名称（对应配置中的 `aliyun.face.db-name`）

### 3. 编译运行后端

```bash
cd backend
mvn clean package -DskipTests
java -jar target/shizong-laoren-1.0.0.jar \
  --DB_PASSWORD=你的MySQL密码 \
  --WECHAT_APP_ID=你的小程序AppID \
  --WECHAT_APP_SECRET=你的小程序Secret \
  --ALIYUN_ACCESS_KEY_ID=你的AccessKeyId \
  --ALIYUN_ACCESS_KEY_SECRET=你的AccessKeySecret
```

或使用环境变量方式：
```bash
export DB_PASSWORD=xxx
export WECHAT_APP_ID=xxx
export WECHAT_APP_SECRET=xxx
export ALIYUN_ACCESS_KEY_ID=xxx
export ALIYUN_ACCESS_KEY_SECRET=xxx
mvn spring-boot:run
```

后端启动后访问：`http://localhost:8080/api`

## 三、微信小程序配置

### 1. 修改后端地址
编辑 `miniprogram/app.js`：
```javascript
baseUrl: 'https://你的域名.com/api',   // 生产环境
// baseUrl: 'http://localhost:8080/api',  // 开发调试
```

### 2. 填写AppID
编辑 `miniprogram/project.config.json`：
```json
{
  "appid": "你的小程序AppID"
}
```

### 3. 在微信开发者工具中：
1. 导入 `miniprogram/` 目录
2. 设置AppID
3. 开发调试时勾选"不校验合法域名"
4. 上线前在微信公众平台配置合法域名

## 四、API接口文档

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /user/login | 微信登录 | 否 |
| GET  | /user/profile | 获取用户信息 | 是 |
| POST | /elderly | 添加老人信息 | 是 |
| GET  | /elderly/my | 我的老人列表 | 是 |
| GET  | /elderly/{id} | 老人详情 | 是 |
| PUT  | /elderly/{id} | 更新老人信息 | 是 |
| DELETE | /elderly/{id} | 删除老人信息 | 是 |
| POST | /face/search | 人脸识别（Base64/URL）| 否 |
| POST | /face/search/file | 人脸识别（文件上传）| 否 |
| POST | /upload/photo | 上传照片 | 是 |

## 五、安全说明

- 身份证号使用 **AES加密** 存储，查询时自动脱敏展示
- JWT Token 有效期 **7天**，过期自动跳转登录
- 老人信息只有 **录入人** 才能修改/删除
- **人脸识别无需登录**，任何人（警察/路人）均可使用扫描功能
- 生产环境请使用 **HTTPS**
